  根因：`configTime(NTP_UTC_OFFSET=8*3600,0,...)` 生成的 TZ 字符串在某些 ESP32 core 对正偏移解析失败，

  `setenv("TZ","CST-8",1); tzset();`，并补 `#include <stdlib.h>`。编译通过（Flash 22.8%/RAM 24.8%）。
  需在真机刷机确认时间显示为北京时间。

- **英文笔画不完整根因（已诊断）**：用户报告"当前界面显示的英文存在笔画不完整"。
  诊断脚本 `firmware/tools/diagnose_ascii.py` + `ascii_compare_png.py` 实证：
  - **旧 inksight.site 后端**用 `INKSIGHT_FONT_ENGINE=bitmap` 优先加载 **PCF 预栅格化位图字体**
    （Inter/Lora/NotoSerifSC，9/10/11/12/13 px，每个像素都手工 hint 过），英文笔画完整。
    fallback 才用 TTF @12-14px（足够像素预算，stroke 能落 ≥1px）。
  - **新固件**的 ascii16 由 `gen_cjk16.py` 用 **NotoSerifSC TTF @16px + fontmode="1" mono** 渲染。
    NotoSerifSC 是 CJK 衬线字体，其 Latin 字形是次要设计 + 16px mono 栅格化时 FreeType hinting
    无法保证细笔画（如 'e' 横、't' 横、'r' 弯钩、'i' 点、's' 曲线）落到 ≥1px → 大量字符视觉断笔。
  - 此外 reminder time / pagination 仍用**老的 5×7 `getGlyph()` 字体**（`drawAsciiText` 路径），
    该字体本身就是 5 列宽，缩放后也不清晰。
  - 修复方向（待用户拍板）：① 用 Inter/Lora TTF 替换 NotoSerifSC 渲 ASCII；② 或在 PCF 工具链
    上把 PCF 烧成 cjk16.h 的 ascii16 部分（用 bdftopcf/bdf2ttf 转换后脚本提取）；③ 或
    字体大小提到 24px 后再 mono（但占 Flash）。
  证据：`firmware/tools/ascii_compare.png`（三行对比，肉眼可见 row1 断笔）和 `ascii_diagnosis.html`。
