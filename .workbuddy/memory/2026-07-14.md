# 2026-07-14 工作日志

- **时区修复（东八区）**：用户反馈设备显示 UTC（7/13 17:54）而非北京时间（7/14 01:54），差 8h。
  根因：`configTime(NTP_UTC_OFFSET=8*3600,0,...)` 生成的 TZ 字符串在某些 ESP32 core 对正偏移解析失败，
  `localtime_r` 回退 UTC。修复：`network.cpp::syncNTP()` 在 configTime 前后显式
  `setenv("TZ","CST-8",1); tzset();`，并补 `#include <stdlib.h>`。编译通过（Flash 22.8%/RAM 24.8%）。
  需在真机刷机确认时间显示为北京时间。
