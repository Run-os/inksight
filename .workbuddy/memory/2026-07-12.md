## 烧录后反馈的功能异常（第二轮修复）
用户烧录后屏亮，但三个问题：①按键无响应/不能切界面；②电量固定0%；③上电约15s才显示。

### 根因与修复（已改，待重新编译验证）
- **按键**：`config.h` 原 `PIN_CFG_BTN=0`（撞 BOOT）。官方引脚图确认 Waveshare ESP32-S3-RLCD-4.2 的 KEY 键在 **GPIO18**，BOOT 在 GPIO0。已改为 `PIN_CFG_BTN=18`。修改后 KEY 键接管配置（长按=进配网 portal），BOOT 回归下载模式专用。固件短按 CFG_BTN 本就"no action"（main.cpp:2293），长按进 portal（main.cpp:2283）——没有"切界面"绑定。
- **电量0%**：两处错。①分压：官方为 **3倍**（wiki + 示例 `0.001*tage*3`），我原写成2倍；②API冲突：原代码 `analogRead()`(Arduino) 与 `esp_adc_cal`(ESP-IDF) 混用抢同一 ADC 易读0。改为 RLCD 专用分支，用 ESP-IDF `adc_oneshot`+`adc_cali`（ADC1_CH3=GPIO4, ADC_ATTEN_DB_12, ADC_BITWIDTH_12），乘3.0。电量显示由设备 heartbeat 上报电压→服务器渲染到图片，故修好 readBatteryVoltage 即恢复。
- **15s启动延迟**：非RLCD bug，是固件固有流程：delay(3000)+WIFI_TIMEOUT(15s)+fetchBMP+NTP(getLocalTime 5s，且在首次 smartDisplay 之前 main.cpp:1125先于1127)。已向用户解释，并提出优化方案（早期 splash / NTP 后置以省~5s），待用户确认是否要改共享启动流程。
- 关键官方引脚参考（Waveshare ESP32-S3-RLCD-4.2）：GPIO0 BOOT、GPIO4 BAT ADC(3x)、GPIO5 DC、GPIO11 SCK、GPIO12 MOSI、GPIO13/14 I2C、GPIO18 KEY、GPIO40 CS、GPIO41 RST、GPIO46 功放使能。电池18650(2.5~4.2V)。


## 第三轮：电池无法开机 / 开机延迟（已改并编译通过）
用户反馈（屏已亮）：①仅长按电源键可关机；②外接电源通电约15s自动开机亮屏；③电池供电：关机后接电池等15s无显示，关机态长按电源键无法开机。

### 根因分析
- 固件"关机"= `enterDeepSleep()` 深度睡眠（非真断电）。唤醒源原仅有 `PIN_CFG_BTN`(GPIO18) + 定时器。
- 外部供电能自动开机：ESP32-S3 的 VBUS 上电会让 EN 复位→冷启动（约15s固件流程），与唤醒源无关。
- 电池下无法开机=两类可能：(a)用户按的是 BOOT(GPIO0)，而它**不是**唤醒源→深度睡眠里按它无效；(b)电池根本没给板子供电（PMIC/电池座/极性）→固件无能为力。
- 15s = delay(3000)+WiFi+fetchBMP+**syncNTP()(在首次刷屏前，最多阻塞5s)**。

### 修复（已改 main.cpp，编译通过 SUCCESS）
1. **唤醒源增加 BOOT(GPIO0)**：`enterDeepSleep()` 里除 `PIN_CFG_BTN`(GPIO18) 外，再 `rtc_gpio_pullup_en(GPIO0)` + 并入 `wakeMask`，LOW 唤醒。sleep 入口两侧都检测"仍被按住则仅定时器唤醒"，避免立即回唤醒。这样电池下按 BOOT 或 KEY 任一键都能唤醒。
2. **开机延迟缩短**：RLCD 初始 `delay(3000)`→`delay(800)`；`syncNTP()` 从首次 `smartDisplay()` 之前移到之后（屏先亮，NTP 后台同步，省约5s）。其它板型保持 3000ms 不变。

### 待用户验证 / 区分硬件还是固件
- 电池单供、设备关机态：长按 **KEY(GPIO18)** 约2-3s 松开，看 RGB(GPIO48) 或屏是否有反应：
  - 有反应 → 之前是深度睡眠、按的是 BOOT 而非唤醒源，本次 GPIO0 唤醒修复已解决；
  - 完全无反应 → 大概率电池未真正供电（电池座未插紧/极性/电池耗尽/PMIC 电池通路问题），属硬件，固件改不了。需测电池座电压确认。
- 注：长按 CFG_BTN(GPIO18) 在固件里是"进配网 portal"（屏变空白但设备仍运行、耗电），不是真关机；真关机是刷新周期结束后的自动深度睡眠或按住 BOOT 触发的断电。
